# Changelog

Все значимые изменения в проекте будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.0] - 2026-01-06

### Добавлено
- Базовая структура расширения для Chrome (Manifest V3)
- OAuth 2.0 авторизация через Google
- Загрузка и анализ писем из Gmail
- Группировка писем по отправителям
- Подсчет статистики (всего писем, непрочитанных, отправителей)
- Поиск ссылок для отписки в заголовках писем
- Массовое удаление писем от выбранного отправителя
- Белый список для защиты важных отправителей
- Поиск и фильтрация отправителей
- Фильтр "только с непрочитанными"
- Современный UI с адаптивным дизайном
- Модальные окна подтверждения перед удалением
- Прогресс-бар для длительных операций
- Кэширование результатов анализа
- Настройки пользователя
- История действий

### Технические детали
- Background Service Worker для работы с Gmail API
- Wrapper для Gmail API с обработкой ошибок и rate limiting
- Парсер для анализа писем и извлечения метаданных
- Storage Manager для работы с chrome.storage.local
- Обработка ошибок 401, 403, 429, 500
- Retry логика с exponential backoff
- Batch requests для оптимизации запросов
- Виртуализация списков (готово к реализации)
- Debounce для поиска (300ms)

### Документация
- README.md с описанием проекта
- SETUP.md с инструкцией по настройке
- .cursorrules с правилами разработки
- Комментарии в коде на русском языке
- JSDoc для публичных функций

### Известные проблемы
- Иконки расширения пока не созданы (используются заглушки)
- Виртуализация списков не реализована (будет добавлена при необходимости)
- Отписка работает только через открытие ссылки в новой вкладке
- Нет возможности отмены удаления (письма удаляются навсегда)

## Версионирование

- **MAJOR** версия увеличивается при несовместимых изменениях API
- **MINOR** версия увеличивается при добавлении новой функциональности с обратной совместимостью
- **PATCH** версия увеличивается при исправлении багов с обратной совместимостью
